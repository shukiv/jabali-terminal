# Jabali Terminal

Standalone addon for Jabali Panel that offers an in-panel **root shell** via
[xterm.js](https://xtermjs.org/). Python daemon runs as root, listens on a
unix socket, bridges WebSocket ↔ PTY. A Filament plugin ships the admin page
and auth flow.

> **Security-critical**: every change that affects **session auth, PTY spawn,
> or audit logging** requires a code-review note explaining why it's safe.
> **No exceptions.** This addon hands root to an authenticated admin over a
> browser — the blast radius is total if auth or audit regresses.

> **Cross-repo context**: sibling of `~/projects/jabali-security` and driven by
> the addon machinery in `~/projects/jabali`. See
> `~/projects/jabali/plans/jabali-terminal-addon.md` for the full construction
> plan and SEC-REV findings.

## Stack

- Python 3.12+, asyncio
- `websockets` for the WS server
- `aiohttp` for the unix-socket HTTP control plane
- `cryptography` for HMAC + nonce signing
- `aiosqlite` for the persistent nonce store (SEC-REV-1)
- `pytest` + `pytest-asyncio` for tests
- xterm.js (+ `xterm-addon-fit`, `xterm-addon-web-links`) in the panel, bundled
  via Vite; no CDN (CSP forbids it)

## Conventions

- Config format: `KEY="value"` shell-style (matches jabali-security)
- Config path: `/etc/jabali-terminal/jabali-terminal.conf`
- Data path: `/var/lib/jabali-terminal/` (holds `nonces.db`)
- Log path: `/var/log/jabali-terminal/sessions/<id>.log` (+ `.sig` after close)
- Runtime socket: `/run/jabali-terminal/jabali-terminal.sock` (root:www-data, 0660)
- Never expose a TCP port — unix socket only; nginx bridges WS → socket
- Never pass the WS token as a URL query param (SEC-REV-2) — use the
  handshake-challenge flow
- Never commit HMAC secrets — generated by `install.sh`
- Never `sed` `AdminPanelProvider.php` — use `class_exists()` autodiscovery
  (learned from jabali-security rollout, 2026-04-12)
- Use `escapeshellarg()` when the panel shells out

## Security Non-Goals

- **Not** providing per-user terminals — admin only
- **Not** providing file upload through the terminal
- **Not** providing session replay/recording playback — text logs only

## Running (development)

```bash
# daemon
cd daemon
make test          # unit + integration
make lint          # ruff
make typecheck     # mypy

# install to a throwaway LXC (destructive — runs as root)
sudo bash install.sh
sudo bash install.sh --uninstall
```
